# Сопутствующая фурнитура на карточке ручки

Документ описывает текущую реализацию блока «Сопутствующая фурнитура» на странице товара. Логика и UI — **макет для дальнейшей доработки**.

## Назначение

На карточке товара из категории **Фурнитура → Ручки** под основным блоком показываются подобранные товары из других подкатегорий фурнитуры (фиксаторы, защёлки, петли), чтобы клиент мог сразу добавить комплект в корзину.

## Когда блок показывается

Блок рендерится **только если** в ответе API есть хотя бы один сопутствующий товар.

На бэкенде подбор запускается только для товара, у которого:

- корневая категория: slug `fittings` (Фурнитура);
- подкатегория: slug `ручки`;
- в `products.attrs` заполнены **Производитель** и **Артикул цвета фурнитуры**.

Если производитель или артикул цвета пустые — возвращаются пустые списки, блок на витрине не появляется.

Для **фиксаторов** дополнительно требуется заполненное поле **Розетка** на текущей ручке. Без него группа фиксаторов не подбирается (защёлки и петли могут показаться, если для них есть совпадения).

## Правила подбора

Значения для сравнения берутся с **текущей ручки** из `products.attrs` (сырые значения JSONB, без единиц измерения).

| Группа      | Подкатегория (slug) | Кол-во | Поля совпадения |
|-------------|---------------------|--------|-----------------|
| `fixators`  | `фиксаторы`         | 1      | Производитель, Артикул цвета фурнитуры, Розетка |
| `latches`   | `защелки`           | 2      | Производитель, Артикул цвета фурнитуры |
| `hinges`    | `петли`             | 3      | Производитель, Артикул цвета фурнитуры |

Дополнительно:

- из выдачи исключается текущий товар;
- уже подобранные id исключаются из следующих групп (без дублей между группами);
- в каждой группе выбор **случайный** (`ORDER BY RANDOM()`);
- только активные товары (`is_active = TRUE`);
- только товары с хотя бы одним непустым фото (то же правило, что у витрины каталога).

Сравнение строк: `LOWER(TRIM(...))` — регистр и пробелы по краям не важны.

## Атрибуты

Коды полей в `attrs` **не захардкожены по русским названиям**. Они резолвятся из `attribute_definitions` по имени (нормализация как в `attributeSemantics.normalizeAttrNameKey`):

| Логическое поле | Имя в справочнике атрибутов | Fallback code |
|-----------------|----------------------------|---------------|
| manufacturer    | Производитель              | `manufacturer` |
| colorArticle    | Артикул цвета фурнитуры    | — (обязателен в БД) |
| rosette         | Розетка                    | — (обязателен в БД) |

Если атрибут с нужным именем не найден, соответствующее поле не сматчится.

## Категории

Slug подкатегорий заданы в константах (кириллические slug из админки):

```
fittings          — корень «Фурнитура»
  ручки           — триггер показа блока
  фиксаторы       — 1 товар
  защелки         — 2 товара
  петли           — 3 товара
```

## Поток данных

```
GET /api/products/:id
  → catalogService.getProductById
  → productRepository.getProductById
  → loadRelatedFittingsForHandle (fittingsRelated.js)
  → поле relatedFittings в JSON
  → normalizeProductData (normalizers.ts)
  → ProductPage: RelatedFittingsSection
```

### Формат `relatedFittings` в API

```json
{
  "fixators": [
    { "id": 1, "sku": "...", "name": "...", "price": 1190, "image": "...", "subcategory": "Фиксаторы", "group": "fixators" }
  ],
  "latches": [ /* до 2 */ ],
  "hinges": [ /* до 3 */ ]
}
```

На фронте массивы **склеиваются в один список** в порядке: фиксаторы → защёлки → петли.

## UI (витрина)

Файл: `src/app/product/[id]/page.tsx`

- Секция: заголовок «Сопутствующая фурнитура», подзаголовок с пояснением про критерии подбора.
- Сетка: `grid-cols-1` / `sm:grid-cols-2` / `lg:grid-cols-3` — до **3 карточек в ряд** на широком экране, без разбивки по подкатегориям.
- Карточка (`RelatedFittingCard`):
  - белый фон (`bg-white`);
  - фото + ссылка на `/product/:id`;
  - название, подкатегория (или SKU), цена;
  - **иконка корзины** поверх фото (правый нижний угол, круглая кнопка 32×32);
  - клик по иконке → `cartStore.addItem`, уведомление «Товар добавлен в корзину».

Блок расположен **под** основной карточкой товара и **над** блоком «Комплектующие в этом цвете» (погонаж по `pogonazh_id` — отдельная, не связанная логика).

## Идеи для следующих итераций

- Убрать `RANDOM()` — фиксированный порядок (цена, популярность, `sort_order`).
- Показывать заголовки подгрупп (Фиксаторы / Защёлки / Петли) или бейдж на карточке.
- Подбор при пустой розетке: fallback без этого поля или скрывать только фиксаторы.
- Отдельный endpoint вместо расширения `getProductById`.
- Кэширование подбора по ключу `(manufacturer, colorArticle, rosette)`.
- Админ-настройка лимитов и списка подкатегорий без правки кода.

## Отличие от `accessories`

| | `relatedFittings` | `accessories` |
|--|-------------------|---------------|
| Триггер | Ручка в `fittings/ручки` | Любой товар с `pogonazh_id` |
| Связь | Атрибуты + подкатегория | Пересечение `pogonazh_id` |
| Типичный кейс | Фурнитура к ручке | Погонаж к двери |
| UI | Карточки-сетка | Таблица со степпером количества |
